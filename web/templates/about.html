Data

The bicycle parking data and the crime data are both provided by the City of San Francisco.

Technology

Python is used to read and munge the data. It is then stored in a PostgreSQL database. Geospatial support from PostGIS makes working with the geographic data straightforward and fast.

Algorithm

For each bike rack, the number of nearby bike thefts are calculated. Since crime may move as the city changes, this number is weighted so that more recent thefts are weighted more heavily.

More bikes will be stolen where more bikes are parked. But there will also be more bike racks where more bikes are parked. So the algorithm divides the number of nearby bicycle thefts by the number of nearby bicycle racks to account for this effect.

To recommend a parking spot, the app computes the distance to and the normalized theft rate at all nearby racks. Both of these numbers are scaled and combined to produce a single number that is used to rank the parking spots.

By changing the scaling, the algorithm can favor closer parking or safer parking. The default option tries to strike a good balance.

App

The app is served up using the Flask framework for Python. It makes extensive use of the Google Maps API. Hosting is provided by Amazon Web Services. The styling is done with Twitter Bootstrap.
